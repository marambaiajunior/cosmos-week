# Checklist antes de pedir nova revisão no AdSense

## 1. Deploy

- [ ] Substituí os arquivos no repositório.
- [ ] Fiz commit e push.
- [ ] Aguardei o GitHub Pages publicar a nova versão.
- [ ] Abri o site em janela anônima.

## 2. URLs obrigatórias

Verifique manualmente:

- [ ] https://www.cosmosweek.com/
- [ ] https://www.cosmosweek.com/ads.txt
- [ ] https://www.cosmosweek.com/sitemap.xml
- [ ] https://www.cosmosweek.com/guias/
- [ ] https://www.cosmosweek.com/politica-de-privacidade.html
- [ ] https://www.cosmosweek.com/contato/
- [ ] https://www.cosmosweek.com/sobre/
- [ ] https://www.cosmosweek.com/padroes/

## 3. Sinais editoriais

- [ ] A home mostra links para guias originais.
- [ ] A navegação principal tem Início, Arquivo, Guias, Sobre, Padrões e Contato.
- [ ] A política de privacidade menciona Google AdSense, cookies de publicidade e opção de desativar anúncios personalizados.
- [ ] Não há banner flutuante “Sua marca aqui”.
- [ ] Não há placeholder de anúncio comercial visível antes da aprovação.

## 4. AdSense

- [ ] O painel do AdSense mostra `ads.txt` como autorizado.
- [ ] O site cadastrado no AdSense é `cosmosweek.com`.
- [ ] O domínio principal redireciona corretamente para a versão canônica usada pelo site.
- [ ] A caixa “Confirmei que corrigi os problemas” foi marcada.
- [ ] Cliquei em “Pedir revisão”.

## 5. Depois do envio

- [ ] Não reenviar revisão repetidamente sem mudanças.
- [ ] Continuar publicando conteúdo autoral nos guias.
- [ ] Evitar publicar muitas notícias reescritas de terceiros de uma só vez.
- [ ] Priorizar artigos com análise própria, contexto, metodologia e limites da evidência.
